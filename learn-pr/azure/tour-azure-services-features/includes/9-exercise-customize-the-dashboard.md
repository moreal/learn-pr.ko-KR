이 연습에서는 사용자 지정 대시보드를 만들고 구성합니다.

## <a name="create-a-new-dashboard"></a>새 대시보드 만들기

1. Azure Portal에서 **새 대시보드** 단추를 클릭합니다.

2. **내 대시보드** 상자에서 이름을 **고객 대시보드**로 변경합니다.

## <a name="add-and-configure-the-clock-tile"></a>시계 타일 추가 및 구성

1. 타일 갤러리에서 시계를 작업 영역으로 끕니다. 사용 가능한 공간의 오른쪽 위에 놓습니다.

2. **시계 편집** 블레이드에서 **태평양 표준시(미국 및 캐나다)** 로 위치를 변경합니다.

3. **시간 형식**에서 **24시간**을 클릭합니다.

4. **Done**을 클릭합니다.

5. 이전 네 단계를 반복하되 **동부 표준시(미국 및 캐나다)** 를 선택합니다. 이제 각각 서부 해안 시간과 동부 해안 시간을 보여 주는 시계가 두 개가 표시됩니다.

## <a name="resize-a-tile"></a>타일 크기 조정

1. **타일 갤러리** 창에서 **모든 리소스**를 클릭하고 새 대시보드 작업 영역 왼쪽 위에 해당 타일을 놓습니다.

2. 해당 타일을 클릭하고, 줄임표를 마우스 오른쪽 단추로 클릭한 후, **6x6**을 클릭합니다.

3. 타일의 오른쪽 아래 회색 모서리를 클릭하고 타일 크기를 세로 3.5, 가로 6으로 조정합니다. 모서리를 놓으면 4x6으로 크기가 조정됩니다.

4. 타일 갤러리에서 **리소스 그룹** 타일을 클릭하여 작업 영역으로 끌어 옵니다. **모든 리소스** 타일 아래에 놓습니다.

5. 타일 갤러리에서 **Service Health** 타일을 클릭하여 작업 영역으로 끌어 옵니다. **모든 리소스** 타일 오른쪽에 놓습니다.

6. 계속 다음 타일을 추가하고 적합하게 다시 정렬합니다.

    * 도움말 + 지원
    * 빠른 작업
    * Marketplace
    * 새로운 기능

7. 이러한 타일을 추가한 후에는 **사용자 지정 완료**를 클릭합니다. **고객 대시보드** 대시보드가 표시됩니다.

## <a name="clone-a-dashboard"></a>대시보드 복제

이제 다른 고객을 위해 아주 유사한 대시보드를 만들려고 합니다.

1. **복제** 단추를 클릭합니다.

2. 대시보드를 **고객 대시보드 복제본**에서 **Azure AD 관리 대시보드**로 이름을 바꿉니다.

3. **리소스 그룹** 타일에서 휴지통 아이콘을 클릭하여 이 타일을 삭제합니다.

4. 타일 갤러리에서 다음 타일을 추가합니다.

    * 조직 ID
    * 사용자 및 그룹
    * 사용자 활동 요약
    * Azure AD 관리 센터 시작

5. 필요에 따라 타일 위치를 변경하고 **사용자 지정 완료**를 클릭합니다.

## <a name="share-a-dashboard"></a>대시보드 공유

이제 이 대시보드를 다른 사용자가 사용할 수 있게 하려고 합니다. 이렇게 하려면 다음 단계를 수행하세요.

1. Azure AD(Azure Active Directory) 관리 대시보드가 선택되어 있는지 확인한 후 **공유**를 클릭합니다.

2. **공유 및 액세스 제어** 블레이드에서 **Publish to the 'dashboards' resource group**(‘대시보드’ 리소스 그룹에 게시)이 선택되어 있는지 확인합니다.

3. **위치**를 해당 지역에 적합한 위치로 설정합니다. 일반적으로 이 값은 가장 가까운 데이터 센터로 기본 설정됩니다.

4. **게시**를 클릭하고 **공유 + 액세스 제어** 블레이드를 닫습니다.

5. **Azure AD 관리 대시보드**를 클릭하고 **고객 대시보드**를 선택합니다.

    **모든 리소스**에 공유 대시보드 리소스가 표시되고 **리소스 그룹**에도 대시보드 리소스 그룹이 표시됩니다.

6. 1단계에서 3단계를 반복하여 고객 대시보드를 공유합니다.

## <a name="edit-a-dashboardjson-file"></a>dashboard.json 파일 편집

대시보드 파일을 다운로드하고 편집하는 방법을 표시하려면 다음 단계를 수행하세요.

1. **다운로드**를 클릭합니다.

2. Windows 탐색기를 열고 다운로드 폴더로 이동합니다.

3. *Customer Dashboard.json* 파일을 찾아 두 번 클릭합니다.

4. 파일 편집기에서 *ClockPart* 텍스트를 찾습니다.

5. 첫 번째 표시된 ClockPart에서 이전 **rowSpan** 값을 1로 변경합니다.

6. 두 번째 표시된 Clockpart에서도 이전 **rowSpan** 값을 1로 변경합니다.

7. 두 번째 표시된 Clockpart에서 Y 값을 2에서 1로 변경합니다.

8. *Customer Dashboard.json* 파일을 저장하고 코드 편집기를 닫습니다.

9. Azure 대시보드에서 **업로드**를 클릭합니다.

10. **열기** 대화 상자에서 다운로드 폴더를 찾아 *Customer Dashboard.json*을 두 번 클릭합니다.

    시계가 한 행 높게 크기 조정되고 아래 시계가 한 행 위로 이동했는지 확인하세요.

## <a name="select-a-shared-dashboard"></a>공유 대시보드 선택

작은 시계를 좋아하지 않는다는 것을 알게 되어 고객 대시보드의 이전 공유 버전으로 돌아가려고 합니다. 파일을 편집하고 다시 업로드하거나 공유 버전에 액세스하여 이전 공유 버전으로 돌아갈 수 있습니다. 이렇게 하려면 다음 단계를 수행하세요.

1. **고객 대시보드** 옆의 아래쪽 화살표를 클릭합니다.

2. **모든 대시보드 찾아보기**를 클릭합니다.

3. **모든 대시보드** 블레이드의 **유형**에서 **공유 대시보드**를 선택합니다.

4. **고객 대시보드**를 클릭합니다.

5. **모든 대시보드** 블레이드를 닫습니다.

    시계가 원래 크기로 돌아갔는지 확인하세요.

## <a name="switch-to-full-screen"></a>전체 화면으로 전환

1. **고객 대시보드** 옆의 아래쪽 화살표를 클릭합니다. 

    옆에 공유 기호가 없는 다른 고객 대시보드가 있습니다. 해당 버전의 고객 대시보드를 클릭하면 시계가 다시 작아집니다.

2. 공유 고객 대시보드로 다시 전환합니다.

3. **전체 화면** 단추를 클릭합니다. 

    브라우저 메뉴와 막대가 모두 사라졌는지 확인하세요.

4. **전체 화면 끝내기**를 클릭하여 일반 화면으로 돌아갑니다.

## <a name="unshare-a-dashboard"></a>대시보드 공유 해제

특정 공유 대시보드를 선택할 수 없도록 하려면 해당 대시보드를 공유 해제할 수 있습니다. 대시보드를 공유 해제하려면 다음 단계를 수행하세요.

1. **공유 해제** 단추를 클릭합니다. **공유 + 액세스 제어** 블레이드가 표시됩니다.

2. **게시 취소** 단추를 클릭합니다.

3. 확인 메시지 상자에서 **확인**을 클릭합니다.

4. **고객 대시보드** 옆의 아래쪽 화살표를 클릭합니다.

5. **모든 대시보드 찾아보기**를 클릭합니다.

6. **모든 대시보드** 블레이드의 **유형**에서 **공유 대시보드**를 선택합니다.

    사용 가능한 대시보드 목록에 **고객 대시보드**가 더 이상 표시되지 않습니다.

7. **모든 대시보드** 블레이드를 닫습니다.

## <a name="delete-a-dashboard"></a>대시보드 삭제

1. **Azure AD 관리** 대시보드가 선택되어 있는지 확인합니다.

2. **삭제** 단추를 클릭합니다.

3. **확인** 메시지 상자에서 확인란을 선택하여 이 대시보드를 더 이상 표시하지 않음을 확인하고 **확인**을 클릭합니다.

## <a name="reset-a-dashboard"></a>대시보드 다시 설정

1. **고객 대시보드**가 선택되어 있는지 확인합니다.

2. **편집**을 클릭합니다.

3. 작업 영역을 마우스 오른쪽 단추로 클릭하고 **Reset to default state**(기본 상태로 다시 설정)를 클릭합니다.

4. **Reset dashboard to default state**(대시보드를 기본 상태로 다시 설정) 메시지 상자에서 **예**를 클릭합니다.

    고객 대시보드가 해당 기본 타일로 다시 설정되었는지 확인하세요.

5. **사용자 지정 완료**를 클릭합니다.

6. 포털 오른쪽 위에 있는 자신의 이름을 클릭합니다.

7. **로그아웃**을 클릭합니다.

8. 브라우저를 닫습니다.

## <a name="summary"></a>요약

대시보드를 만들고, 편집하고, 공유하고, **.JSON** 파일로 변경하고, 공유를 해제하고, 마지막으로 기본 상태로 다시 설정했습니다. 이제 대시보드가 얼마나 강력한 도구이며, 어떻게 하면 대시보드로 조직 내의 다양한 역할에서 사용할 효율적인 인터페이스를 만들 수 있는지 이해합니다.
